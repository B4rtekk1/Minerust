## Code Organization Principles

### Module Hierarchy

```
minerust (binary root)
├── app/          ← Entry point for game loop
├── core/         ← Fundamental data structures
├── render/       ← GPU rendering pipeline
├── world/        ← Procedural generation
├── player/       ← Camera and movement
├── multiplayer/  ← Networking
├── ui/           ← User interface
├── utils/        ← Configuration and utilities
└── settings/     ← Settings files
```

### Dependency Graph

```
app ──────→ render ─────────→ core
  ├──────→ player              ↑
  ├──────→ world ──────────────┤
  ├──────→ ui                  ↑
  ├──────→ multiplayer ────────┤
  └──────→ utils

Principles:
✓ Lower layers don't depend on higher layers
✓ Clear data flow direction
✓ Circular dependencies avoided
✓ Loosely coupled modules
```

### File Naming

```
mod.rs              - Module declaration and public API
type_name.rs        - Primary type definition
system_name.rs      - System implementation
utilities.rs        - Helper functions
constants.rs        - Global constants
```

## Core Systems

### 1. Rendering Pipeline

**Frame Rendering Order:**
```
1. Input Processing
   └─ Keyboard/mouse input

2. Game Update
   ├─ Player physics
   ├─ Chunk loading
   └─ Entity updates

3. Shadow Pass
   └─ Render to shadow maps (CSM)

4. Depth Prepass (Optional)
   └─ Early Z rejection

5. Terrain Pass
   ├─ Frustum culling
   ├─ Indirect draw commands
   └─ Main geometry

6. Water Pass
   └─ Special water effects

7. Composite/Post-Process
   ├─ Bloom
   ├─ Tone mapping
   └─ Color correction

8. UI Pass
   ├─ Text rendering
   ├─ HUD elements
   └─ Menus (if open)

9. Present
   └─ Display to monitor
```

**GPU Buffer Architecture:**
```
Unified Vertex Buffer (560 MB)
  └─ Subdivided into chunks
     └─ Each chunk: [vertex_offset, num_vertices]

Unified Index Buffer (256 MB)
  └─ Subdivided into chunks
     └─ Each chunk: [index_offset, num_indices]

Indirect Buffer
  └─ Draw commands generated by compute shader
```

### 2. World Generation

**Generation Algorithm:**
```
Seed → Noise Functions → Biome Lookup → Terrain Height
  ↓
Block Placement → Cave Carving → Ore Generation
  ↓
Structure Placement (trees, buildings)
  ↓
Final Chunk
```

**Chunk Loading:**
```
Player Position (64, 100)
  ↓
Calculate Required Chunks
  └─ In radius: RENDER_DISTANCE = 10
  └─ Area: 20×20 = 400 chunks
  ↓
Async Generation (worker thread)
  ├─ Generate terrain
  ├─ Place features
  └─ Build mesh
  ↓
Upload to GPU
  └─ Add to vertex/index buffers
```

### 3. Networking Protocol

**Message Format:**
```
[1 byte: Type] [2 bytes: Size] [N bytes: Data]

Example:
Type=0x05 (BlockUpdate)
Size=0x000B (11 bytes)
Data=[x, y, z, block_type]
```

**Connection Handshake:**
```
Client → ClientHello
       ← ServerHello
       → ClientReady
       ← Ready

Then gameplay communication begins
```

## Performance Optimization

### GPU Optimizations

| Technique | Impact | Trade-off |
|-----------|--------|-----------|
| Frustum Culling | 80% fewer draws | CPU cost |
| Indirect Drawing | Single draw call | GPU culling shader |
| Mesh Caching | No regeneration | Memory usage |
| Texture Atlasing | Single bind | Atlas management |
| Early Z | Less pixel work | Extra geometry pass |
| LOD System | Distance quality | Visual quality |

### CPU Optimizations

| Technique | Impact | Trade-off |
|-----------|--------|-----------|
| Async Generation | No frame drops | Threading complexity |
| Frame Budgeting | Smooth FPS | Slower mesh generation |
| Chunk Pooling | Allocation free | Memory overhead |
| Binary Serialization | Fast I/O | Less human readable |
| Compression | Smaller chunks | CPU cost |

### Memory Optimizations

```
Block Data:
  ✓ Store as enum (1 byte per block)
  ✗ Store as full struct (16+ bytes)
  
Vertices:
  ✓ Pack UV, AO into small fields
  ✗ Use separate buffers for each
  
Textures:
  ✓ Use texture atlas (single bind)
  ✗ Separate texture per block type
```

## Common Issues & Solutions

### Issue: Frame Rate Drops

**Causes:**
- Too many chunks loaded
- Mesh generation stalling
- GPU memory pressure

**Solutions:**
- Reduce render distance
- Check GPU memory usage
- Monitor mesh generation queue

### Issue: Memory Leak

**Debugging:**
```bash
valgrind --leak-check=full ./minerust
```

**Common Causes:**
- GPU resources not freed
- Circular references
- Unbounded collections

### Issue: Rendering Artifacts

**Causes:**
- Z-fighting (depth precision)
- Texture coordinates wrong
- Normals incorrect

**Debug:**
- Show normals as colors
- Show UV coordinates
- Verify frustum calculations

### Issue: Network Lag

**Symptoms:**
- Player position jumps
- Block placement delayed
- Chunk loading slow

**Solutions:**
- Check bandwidth usage
- Monitor message latency
- Enable chunk compression

## Extending Minerust

### Adding a New Block Type

1. Add variant to `BlockType` enum (core/block.rs)
2. Add texture to assets/textures/block/
3. Update atlas_map_structure.json
4. Add generation rules (world/generator.rs)
5. Update biome spawning (world/terrain.rs)
6. Test in game

### Adding a New Biome

1. Add variant to `BiomeType` enum (core/biome.rs)
2. Define biome properties (temperature, humidity, etc.)
3. Implement generation algorithm
4. Add structure spawning (trees, villages, etc.)
5. Test world generation

### Adding a New Render Pass

1. Create pass module (src/render/passes/my_pass.rs)
2. Implement `RenderPass` trait
3. Create pipeline and bind groups
4. Integrate into render.rs
5. Test rendering

### Adding Multiplayer Feature

1. Define message type (multiplayer/protocol.rs)
2. Implement send logic (client.rs or server.rs)
3. Handle received messages
4. Test with multiple clients
